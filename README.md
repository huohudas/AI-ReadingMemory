# 📖 微信读书 AI 前情提要功能 Demo

> 课程作业 Demo · 产品经理课程 · 技术栈：Streamlit + DeepSeek API

## 🎯 产品背景

现代人阅读高度碎片化，长篇小说或专业书籍常因中断数日后难以续读，导致弃书率居高不下。本 Demo 模拟了一个「AI 驱动前情提要」功能，帮助用户在重新打开书籍时，快速找回记忆与阅读状态。

## ✨ 功能演示

### 核心交互流程

1. **触发提示**：模拟用户距上次阅读超过 3 天，顶部自动出现提示横幅
2. **生成回顾**：点击按钮，AI 基于前文内容生成专属前情提要
3. **千人千面**：结合用户历史划线记录，AI 推断阅读偏好，输出个性化内容
4. **CoT 透出**：右侧面板实时展示 AI 的推理链路（Chain of Thought）

### Demo 界面

| 区域 | 说明 |
|------|------|
| 左侧侧边栏 | 后台控制台：章节切换、时间模拟、划线管理 |
| 中间手机模拟器 | 模拟微信读书 App 内正文阅读界面 |
| 右侧面板 | AI 分析链路可视化，展示系统推理过程 |

## 🛠 技术实现

- **前端框架**：Streamlit（Python）
- **AI 模型**：DeepSeek Chat（`deepseek-chat`）
- **核心能力**：长文本摘要 + 用户偏好推断 + Chain of Thought 透出
- **个性化机制**：用户历史划线 → Prompt 注入 → 千人千面输出

## 🚀 本地运行

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 配置 API Key（在项目根目录创建 api_key.txt）
echo "你的DeepSeek API Key" > api_key.txt

# 3. 准备书籍文件（txt 格式，章节以"第X章"开头）
# 已包含示例文件 book1.txt

# 4. 启动应用
streamlit run app.py
```

核心功能点：
- 渐进式唤醒卡片（三层级设计，轻打扰）
- 防剧透机制（动态锚定阅读进度）
- 私域数据赋能（历史划线 × AI 情绪唤醒）
